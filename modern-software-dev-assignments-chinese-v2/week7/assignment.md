# 第7周 — 使用Gitee和AI脚本探索代码审查

## 作业概述
在这个作业中，你将在更高级的代码库上练习代理驱动开发和AI辅助代码审查。你将实现`week7/docs/TASKS.md`中的任务，使用测试和手动审查验证你的工作，并将你自己的审查笔记与AI生成的代码审查进行比较。

## 开始使用Gitee代码审查
1. 注册Gitee账户：https://gitee.com/
2. 创建代码仓库用于练习
3. 使用Gitee的Pull Request功能进行代码审查
4. 使用本地AI审查脚本辅助代码质量检查

## 要做什么
使用你选择的AI编码工具（例如Cursor、Copilot、Claude等）实现`week7/docs/TASKS.md`中的任务。

### 对于每个任务：
   1. 创建一个单独的分支。
   2. 使用你的AI工具通过1-shot提示实现任务。
   3. 手动逐行审查更改。修复你注意到的任何问题，并在有帮助的地方添加说明性提交消息。你也可以与同学配对审查彼此的代码，而不是审查自己的更改。
   4. 为任务打开拉取请求（PR）。确保你的PR包含：
      - 问题描述和你的方法。
      - 执行测试的总结（包含命令和结果）以及任何添加/更新的测试。
      - 显著的权衡、限制或后续工作。
   5. 使用本地AI审查脚本在PR上生成代码审查建议。
   6. 在`writeup.md`中记录你的PR结果。

## 交付物
在你的`writeup.md`中，我们需要以下内容：

- 四个PR，每个完成的任务一个，每个包含：
  - 清晰的PR描述
  - 相关提交/问题的链接。
  - PR上可见的本地AI审查脚本生成的评论

- 简要反思，解决以下问题：
  - 你在手动审查中通常做的评论类型（例如，正确性、性能、安全性、命名、测试差距、API形状、UX、文档）。
  - **你的**评论与**本地AI脚本的**生成评论在每个PR中的比较。
  - AI审查何时比你的更好/更差（引用具体示例）
  - 你对未来信任AI审查的舒适度以及何时依赖它们的任何启发式方法。

## 评分标准（总分100分）
- 每个完成的任务20分
  - 每个任务的技术正确性和完整性。
  - 代码质量：可读性、命名、结构、错误处理和测试。
  - 手动审查笔记的深思熟虑程度
  - 本地AI脚本生成的代码审查
- 简要反思20分
  - 你审查与本地AI脚本审查之间有洞察力的比较
  - 对AI审查个人舒适度的描述
